01. Sass介紹

Sass 是CSS的預前處理器，目的在減少CSS的重工，以及更有效率的維護樣式的結構，它需進行編譯(complier)成原生的CSS，瀏覽器看不懂 Sass

Sass 可以使用變數、函數、流程控制、嵌套、繼承、混入等功能，被稱為CSS的程式語言(SassScript)。

02. Sass的資料型態

  1. 1. number:數字有分成有單位或無單位，1,12,133,10px,20rem
  2. 2. string:有引號字串 'Noto Sans TC' 與無引號字串 sans-serif
  3. 3. color:blue,#04a3f9,rgba(0,0,0,0.5),linear-gradient()
  4. 4. boolean(布林值):true,false
  5. 5. null(空值)
  6. 6. list(像陣列): 用半形空格或逗號作為區隔，例如:1.5rem 1rem 0 2rem、10px,10px,20px,20px、Arial, Helvetica, sans-serif
  7. 7. map(物件):類似 JavaScript 的物件

03. 變數

。宣告使用 $，變數名稱的位置是它的作用區域，就像 JS 的 let 宣告，在大括號裡面屬於大括號內的區域變數，寫在外面的全域變數

。變數的底線_和連結線-可以互換使用，$bg_dark和$bg-dark兩者是一樣的

。!default為預設，若該變數前後已經被賦值，會引用套用的值，!default就不會被套用

。使用變數可以維持網站一致性，節省修改成本

05. 嵌套巢狀

。&符號可以表示父層選擇器

。勿過度使用嵌套，考慮用最少的選擇器，選擇到想要的位置，維持在3~4層

。過度使用嵌套，編譯後選擇器會有過長的問題，樣式碼不易閱讀，渲染效率較差，造成CSS檔案容量較大

06. Extend(繼承)%

。 繼承可以使用CSS選擇器或 % 佔位選擇器來命名

。 使用CSS選擇器當作繼承，會被編譯到CSS，%佔位選擇器屬於SCSS的範疇，不會被編譯到CSS

。 @extend 沒有參數可以使用，mixins 有參數可以使用

07. 計算

。 可以做加、減、乘、除、取餘數的計算

。 Sass 的計算與calc()不同，Sass預處理前已經計算好，不會造成瀏覽器的負擔
